# Changelog

## 0.1.0-preview.1 — 2026-07-17

### 改进

- README 改用实际运行中的 Codex 工作区、Chat / Work 双模式、Studio 主题画廊、当前主题及恢复页面截图，不再以主题合成预览图代替产品界面。
- 仓库收拢为 Windows APP 单一交付：移除 macOS、旧的独立脚本启动入口、旧图库和平台说明；换肤引擎作为 Studio 内部组件随 EXE 发布，用户不再需要手动执行脚本。
- 放大 Studio 的默认与最小窗口并新增最大化/还原按钮；修复高 DPI 下“主题画廊”等页标题被固定高度裁切，同时为左侧「主题画廊 / 当前主题 / 启动与恢复 / 关于与安全」补齐真实页面切换、选中状态和对应说明内容。
- 修复主题实际应用完成后 Studio 仍永久显示进度条：引擎结果改由独立临时 JSON 文件返回，不再等待会被 Codex 子进程继承、因而无法关闭的标准输出管道；超时时也只终止桥接进程，不中断已启动的 Codex。
- 识别 Codex 官方“一次性 Windows 权限设置”页面：当官方程序未保留调试端口参数时，Studio 会尽快停止进度条并提示用户完成 UAC 设置；启动失败会自动还原应用前颜色，不再遗留半应用状态。
- 修复官方结构化 Chrome 颜色表与 Miku 内联颜色表同时存在造成的 TOML 重复键；现在会完整备份、替换并恢复颜色、字体与语义颜色子表，Windows 设置按钮不再因配置加载失败而失效。
- 修复“恢复官方”只移除页面注入、却遗留 Miku 浅青色全局外观的问题：每次首次应用前精确保存用户原有的三项颜色配置，重复应用不覆盖备份，恢复时逐项还原并以官方方式重启 Codex。
- 缩短 Studio 应用主题后的阻塞验证等待，并为底层调用增加有限超时，避免 Codex 已经完成换肤后进度条仍长时间运行。
- 放宽 Windows Studio 的默认窗口、页面边距、主视觉、主题卡与操作区间距，并修复主视觉渐变越界平铺造成的深蓝竖条。
- 新增 Windows「Codex Dream Skin Studio」预览版：内置 Miku Aqua 主题画廊、运行状态、一键应用启动、重新应用、验证、恢复与系统托盘入口，并随程序携带当前换肤引擎和主题素材。
- 修复显示 Codex 宠物后透明宠物窗口被主题背景填色的问题：注入器现在只对同时具备主区、侧栏和原生内容标记的完整 Codex 主窗口应用皮肤，并主动清理辅助窗口中残留的主题层。
- 修复顶部 Chat / Work 切换器周期性频闪：主题守护检查改为幂等更新，只在原生模式或页面节点实际变化时调整样式类。
- 单独适配 Codex 工作区主页：避免与 Chat/Work 首页规则重复命中，移除重复副标题，欢迎文案居中，并统一四张快捷卡、输入框及“拉取请求”导航的主题样式。
- 适配 Codex 26.715+ 新版主界面：围绕顶部 Chat / Work 切换、中央欢迎语、单行 Chat 输入框与多行 Work 输入框重新设计 Miku 创作舞台，不再依赖旧版四卡片首页结构。
- Chat 与 Work 的原生动态欢迎标题和主题副标题统一在主视觉中居中显示。
- 调整左侧工作区与导航的字体、字重和颜色层级：工作区使用青绿手写感标题，导航按青、粉、蓝、紫图标体系配色，项目与最近任务使用更稳重的深青蓝正文。
- 新版 Chat / Work 切换器、Work 项目/插件工具条、权限与模型选择继续使用统一的青绿、樱粉与淡紫玻璃样式；普通任务页仍保持原生布局和完整交互。
- 默认 Windows Dream Skin 现在为「初音未来」水蓝粉紫主题：青绿玻璃侧栏、音乐星光、01 识别与专属首页主视觉。
- 使用项目内可维护的矢量主题插画；原生侧栏、建议卡片、项目选择和输入框仍保持真实可交互。
- 补齐普通任务页：左上角工作区品牌、侧栏底部状态区、输入框内的权限/项目选择、模型选择和下拉菜单均采用同一套青绿玻璃卡片样式。
- 修正首页卡片的图标/标题间距、卡片下方大段空白、顶部菜单误胶囊化和侧栏账号双头像问题。
- 四张建议卡片改为与主视觉分离并保留小间距，图标使用固定中心坐标；移除左下角自定义 Miku 头像。

### 说明

- 主题仍通过本机回环 CDP 注入，不修改 WindowsApps、官方应用文件或签名。
